# ЛР7. Задание 3

Тема: методы оптимизации в нейросетевых вычислениях.

Здесь решается задача бинарной классификации на табличных данных COVID-19 Dataset.
Модель по медицинским признакам пациента предсказывает, умер пациент или нет.

Внимание! Датасет можно скачать по ссылке: 
https://www.kaggle.com/datasets/meirnizri/covid19-dataset

Target:

- `DATE_DIED = 9999-99-99` -> `0`, пациент не умер;
- другая дата -> `1`, пациент умер.

## Как запустить

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Положить csv-файл в папку `data`.

По умолчанию код ждёт файл:

```text
data/Covid Data.csv
```

3. Запустить обучение:

```bash
python src/train.py
```

Или указать путь явно:

```bash
python src/train.py --csv "data/Covid Data.csv" --rows 2000
```

Запуск с настройками против переобучения:

```bash
python src/train.py --antioverfitting
```

## Что будет после запуска

В папке `results` появятся:

- `metrics.csv` — метрики для двух оптимизаторов;
- `report.txt` — текстовый отчёт;
- `losses.png` — график изменения функции потерь.

## Что сравнивается

Обучается одна и та же нейросеть двумя способами:

1. `torch.optim.RMSprop` — готовый оптимизатор из PyTorch.
2. `MyRMSprop` — собственная реализация RMSProp, оформленная как PyTorch-оптимизатор.

Внешне подмена почти не заметна: оба оптимизатора передаются в обычный цикл обучения через методы `zero_grad()`, `backward()` и `step()`.

## Предобработка данных

Используются уточнения по пропускам:

- `PREGNANT = 97` для мужчин (`SEX = 2`) заменяется на `2`, то есть "нет";
- `ICU = 97` и `INTUBED = 97` для пациентов с `PATIENT_TYPE = 1` заменяются на `2`, то есть "нет";
- оставшиеся `97`, `98`, `99` в `PREGNANT`, `ICU`, `INTUBED` считаются пропусками;
- в `PNEUMONIA` пропуском считается `99`;
- в остальных булевых медицинских признаках пропуском считается `98`.

После очистки строки с пропусками удаляются.

## Гиперпараметры по умолчанию

```text
rows = 2000
epochs = 25
batch_size = 64
learning_rate = 0.001
RMSProp alpha = 0.99
dropout = 0.2
weight_decay = 0
network = 64 -> 32 -> 1
seed = 42
```

Если включить `--antioverfitting`, применяются настройки против переобучения:

```text
epochs = 6
dropout = 0.35
weight_decay = 0.0001
network = 32 -> 16 -> 1
добавляется dropout после второго скрытого слоя
```

Эти значения можно переопределить вручную:

```bash
python src/train.py --antioverfitting --epochs 8 --dropout 0.4 --weight-decay 0.0005
```

Если после очистки в случайных 2000 строках останется слишком мало объектов или только один класс, увеличьте число строк:

```bash
python src/train.py --rows 5000
```
