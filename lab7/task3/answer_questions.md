# Ответы на вопросы для самоконтроля по заданию 3

## 1. Что такое машинное обучение?

Машинное обучение — это подход, при котором программа не получает все правила явно, а находит закономерности по данным. В этой работе модель получает медицинские признаки пациента и учится предсказывать целевой признак: умер пациент или нет.

## 2. Почему нейросети относятся к глубокому обучению?

Глубокое обучение связано с нейронными сетями, у которых есть несколько слоёв. Каждый слой строит новое представление данных: сначала более простое, потом более сложное. В этой работе используется MLP: несколько полносвязных слоёв и функции активации.

## 3. Какая задача решается?

Решается задача бинарной классификации пациентов с COVID-19. На вход подаются признаки пациента: возраст, пол, тип лечения, наличие пневмонии, диабета, ожирения, астмы, гипертонии и других заболеваний. Целевой признак строится по `DATE_DIED`: если стоит `9999-99-99`, пациент не умер; иначе пациент умер.

## 4. Какая предобработка могла бы улучшить качество?

Качество могли бы улучшить более аккуратная обработка пропусков, балансировка классов, анализ выбросов, подбор признаков, кодирование категориальных признаков через one-hot encoding, подбор архитектуры сети и гиперпараметров. В текущей версии строки для обучения берутся случайно из всего CSV, а не первые `N` строк, потому что исходный файл может быть упорядочен и первые строки дают искажённое соотношение классов. Также добавлена предобработка некоректного заполнения данных в датасете (подробнее в [README.md](./README.md))

## 5. Какая целевая функция используется?

Используется `BCEWithLogitsLoss`, то есть бинарная кросс-энтропия с logits. Для каждого объекта сеть выдаёт одно число. После сигмоиды оно интерпретируется как вероятность класса 1. Loss штрафует модель, если вероятность класса смерти высока для выжившего или мала для умершего.

## 6. Какие метрики используются?

Используются accuracy, precision, recall и F1. Accuracy показывает долю правильных ответов. Precision показывает, какая доля предсказанных смертей действительно была смертью. Recall показывает, какую долю реальных смертей модель нашла. F1 объединяет precision и recall.

## 7. Используется ли early stopping?

В коде early stopping не используется. Для борьбы с переобучением добавлен отдельный режим `--antioverfitting`, где количество эпох заранее уменьшено.

## 8. Что такое переобучение и как с ним бороться?

Переобучение — это ситуация, когда модель хорошо запомнила обучающую выборку, но плохо работает на новых данных. На графике это видно, когда train loss продолжает снижаться, а test loss перестаёт снижаться или начинает расти.

В проекте для борьбы с переобучением добавлен флаг `--antioverfitting`. При его использовании применяются четыре меры:

- уменьшается число эпох: `epochs = 6` вместо `25`;
- увеличивается dropout: `0.35` вместо `0.2`;
- добавляется L2-регуляризация через `weight_decay = 0.0001`;
- уменьшается размер сети: `32 -> 16 -> 1` вместо `64 -> 32 -> 1`.

Без флага `--antioverfitting` запуск остаётся в обычном режиме, чтобы можно было сравнить результат до и после регуляризации.

## 9. Какие гиперпараметры оптимизаторов явно указаны?

В коде явно указаны скорость обучения `lr`, коэффициент сглаживания RMSProp `alpha`, маленькая добавка `eps`, защищающая от деления на ноль, и `weight_decay` для L2-регуляризации. В запуске также задаются размер батча, количество эпох, dropout и размеры скрытых слоёв. Для честного сравнения одни и те же значения передаются в `torch.optim.RMSprop` и в собственный `MyRMSprop`.

## 10. Какие слои есть в нейронной сети?

Используются полносвязные слои `Linear`, функции активации `ReLU` и слой `Dropout`. `Linear` строит линейную комбинацию признаков, `ReLU` добавляет нелинейность, а `Dropout` помогает уменьшить переобучение.

В обычном режиме архитектура такая: `20 -> 64 -> 32 -> 1`, dropout стоит после первого скрытого слоя. В режиме `--antioverfitting` архитектура уменьшается до `20 -> 32 -> 16 -> 1`, а dropout применяется после обоих скрытых слоёв.
